# 飞行前检查清单

**来源：** 2026-03-31 多次错误反思 
**日期：** 2026-03-31 14:00 UTC 
**严重度：** 🔴 高（核心工作流程）

---

## ❌ 错误做法

**问题：** 操作前不检查，凭印象行动

**典型场景：**
1. ❌ 回答配置问题前不搜索记忆
2. ❌ 执行操作前不验证假设
3. ❌ 下结论前不检查所有位置
4. ❌ 不等待提醒就不执行

**后果：**
- 重复犯错
- 浪费少爷时间
- 违背"验证优先于假设"的承诺

---

## ✅ 正确做法

### 飞行前检查清单（每次操作前必读）

**阶段 1：操作前检查（必须完成才能行动）**

- [ ] **搜索相关记忆** - `memory_search` 搜索关键词
- [ ] **检查配置清单** - `config-registry.json` 或 `configured-services-inventory.md`
- [ ] **检查教训文件** - `lessons/` 目录相关教训
- [ ] **验证假设** - 所有假设都已验证
- [ ] **准备回滚方案** - 如果出错如何恢复

**触发词（遇到这些词必须执行检查）：**
- "能不能" / "可不可以" / "是否支持"
- "怎么用" / "如何使用" / "用法"
- 任何工具/技能/配置相关问题

---

### 阶段 2：执行中检查

- [ ] **按清单执行** - 不跳过任何步骤
- [ ] **记录操作** - 记录到 memory/episodic/
- [ ] **验证结果** - 操作后验证是否符合预期

---

### 阶段 3：操作后检查

- [ ] **记录教训** - 有新教训 → 写入 lessons/
- [ ] **更新配置** - 配置变更 → 写入 config-registry.json
- [ ] **记录决策** - 重要决策 → 写入 MEMORY.md
- [ ] **验证回滚** - 确认回滚方案可用

---

## 📋 检查清单模板

```bash
#!/bin/bash
# pre-flight-check.sh - 飞行前检查脚本

QUERY="$1"

# 1. 检查是否包含触发词
if echo "$QUERY" | grep -qE "能不能 | 可不可以 | 是否支持 | 怎么用 | 如何使用"; then
 echo "⚠️ 触发强制记忆搜索"
 
 # 搜索记忆文件
 grep -r "$QUERY" ~/workspace/memory/ 2>/dev/null | head -5
 
 # 检查配置清单
 cat ~/workspace/memory/config-registry.json 2>/dev/null
 
 # 检查教训文件
 ls ~/workspace/memory/lessons/ 2>/dev/null
fi
```
