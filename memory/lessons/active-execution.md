**来源：** 2026-03-31 多次错误反思

**严重度：** 🔴 高（核心认知偏差）

---

## ❌ 错误做法

**问题 1：依赖外部提醒**
- 等用户提醒才去做
- 等心跳 poll 才执行
- 没有形成主动执行的习惯

**问题 2：记忆文件≠真正记住**
- 写在文件里不等于内化
- 没有定期复习
- 没有实际应用

**问题 3：凭印象回答**
- 不先查记忆文件就下结论
- 依赖短期记忆而非文件记录
- 重复犯错

**问题 4：没有检查清单**
- 缺少强制性的前置检查步骤
- 操作前不验证
- 急于下结论

---

## ✅ 正确做法

### 方案 1：强制记忆搜索

**规则：** 回答以下类型问题前，**必须**先搜索记忆文件：

**触发词：**
- "能不能" / "可不可以" / "是否支持"
- "怎么用" / "如何使用" / "用法"
- 任何工具/技能/配置相关问题

**强制步骤：**
1. `memory_search` 搜索相关记忆
2. 检查 `configured-services-inventory.md`
3. 检查 `lessons/` 目录相关教训
4. 然后才能回答

---

### 方案 2：HEARTBEAT 主动执行

**规则：** 每天 09:00 主动执行，不等待心跳 poll

**检查清单：**
```markdown
每天 09:00 (主动执行，不等提醒)：
- [ ] 配置健康检查 (check-config.sh)
- [ ] 邮件检查 (未读邮件)
- [ ] 日历检查 (今日/明日事件)
- [ ] 记忆复习 (根据星期几复习指定文件)
- [ ] 教训复习 (随机 1 个 lessons/ 文件)

每次会话结束前：
- [ ] 今天有没有新教训？→ 写入 lessons/
- [ ] 今天有没有重要决策？→ 写入 MEMORY.md
- [ ] 今天有没有配置变更？→ 写入 config-registry.json
```

---

### 方案 3：教训复习自动化

**流程：**
1. 随机选择 1 个教训文件
2. 读取并理解内容
3. 思考如何应用到今天
4. 记录复习结果到 `reviews/lesson-review.md`

**高优先级教训 (每天复习，直到连续 7 天不犯错)：**
- [ ] `steel-api-test.md` (已连续犯错 3 次)
- [ ] `verify-before-assume.md` (核心认知偏差)
- [ ] `active-execution.md` (本文件)
